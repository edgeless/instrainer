### `scoreState.noteResults` のリアクティビティ（Svelte 5）について
`src/lib/components/ScoreSection.svelte` にて、`$effect` 内で `playerState.currentNoteIdx` などの変更をトリガーにして `renderScore`（経由で `buildStaffRows`/`buildTabRows`）を呼び出しています。
今回の変更で `renderScore` 内部から `scoreState.noteResults` へアクセスするようになりましたが、`renderScore` は `requestAnimationFrame` の非同期コールバックとして実行されるため、Svelte 5の自動トラッキングシステムからは監視されません。

そのため、仮に `currentNoteIdx` が変化したタイミングではなく、その少し後に `noteResults` の評価が確定した場合（例えばフリーモードや遅延評価など）、UI上の色づけが「次の音符へ進んだ時点」まで反映されない可能性があります。
色を即座に反映させるには、以下のように `$effect` ブロック内で明示的にトラッキングを追加することをお勧めします。

```typescript
  $effect(() => {
    void playerState.currentSongKey;
    void playerState.isPlaying;
    void playerState.isRecording;
    void playerState.currentNoteIdx;

    // トラッキングを追加（※ Svelte 5における配列変更のトラッキングのため、
    // 必要に応じて length を参照したり、別のアプローチを取る必要があるかもしれません）
    void scoreState.noteResults.length; 
    
    requestAnimationFrame(renderScore);
  });
```
